# ADR 0001 — 백엔드 인프라로 NCP 채택

- 상태: 채택됨
- 날짜: 2026-06-23

## 맥락

저비용(가능하면 $0) 운영이 목표. 백엔드는 **Java/Spring + MySQL** 사용이 확정 제약.
chipthrone는 AWS 프리티어 + Vercel로 운영 중.

## 검토한 선택지

| 선택지 | 비용 | Java/Spring | MySQL | 결과 |
|---|---|---|---|---|
| AWS 프리티어 (기존 계정) | 12개월 후 과금, EC2 750h는 계정 합산(2대 불가) | ✅ | RDS 한시 | 기각 |
| **AWS 프리티어 2번째 계정** | $0 | ✅ | ✅ | **기각 (약관 위반)** |
| Oracle Cloud Always Free | 영구 $0, MySQL HeatWave 무료 | ✅(ARM) | ✅ | 차선 |
| AWS t4g.small | 2026-12-31까지만 무료 | ✅ | RDS 한시 | 기각 |
| **NCP (네이버 클라우드)** | 크레딧 10만원/3개월, 이후 소액 과금 | ✅ | ✅(Cloud DB for MySQL) | **채택** |

### 왜 AWS 2번째 계정을 기각했나

- AWS Free Tier는 1인 1계정 전제 → 프리티어 반복 수령 목적의 다계정은 AUP 위반.
- AWS는 카드/신원/IP/디바이스로 동일인 연결 가능 → 적발 시 **chipthrone가 올라간 기존 계정까지 정지 위험**(비대칭 리스크).
- 새 계정도 결국 만료 → "프리티어 농사"를 매년 반복해야 함.

### 왜 Oracle이 아니라 NCP인가

- Oracle Always Free가 "영구 $0 + 관리형 MySQL 무료"로 비용만 보면 최선.
- 그러나 사용자가 NCP를 선택. 근거:
  - 타깃이 **한국 퇴직연금 사용자** → 국내 리전 지연시간 최상.
  - 원화 결제, 한국어 콘솔/문서/CS.
  - 금융·공공 클라우드 친화 (연금 도메인 톤과 부합).
  - Oracle A1 인스턴스 재고 이슈/해외 가입 번거로움 회피.

## 결정

백엔드(Spring) + MySQL을 **NCP**에 호스팅. 프론트는 Vercel(무료) 유지.

## 결과 / 트레이드오프

- (+) 국내 사용자 응답 빠름, 국내 운영 편의.
- (−) 영구 무료 아님. 가입 크레딧(10만원/3개월) 소진 후 월 소액 과금 → 운영비 발생 수용.
- 후속: NCP 구체 상품(Micro Server vs Server, Cloud DB for MySQL vs 직접설치)은 별도 결정.
- 비상시 이전 경로: 비용이 부담되면 Oracle Always Free로 이전 가능(수식·코드는 이식성 있게 유지).
