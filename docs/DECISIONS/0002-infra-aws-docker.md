# ADR 0002 — 배포 인프라를 AWS EC2 + Docker/GHCR 로 변경 (RDS 미사용)

- 상태: 채택됨 ([ADR 0001](0001-infra-ncp.md) 의 NCP 결정을 대체)
- 날짜: 2026-06-25

## 맥락

ADR 0001은 NCP를 채택했으나, 이후 사용자의 AWS 계정 상황이 구체화되며 재검토.

- chipthrone.com 이 이미 **사용자의 AWS 계정**에서 t3.micro(프리티어)로 운영 중.
- AWS 프리티어가 2025-07 정책 변경으로 **크레딧 기반**(가입 $100 + 활동 보상 최대 $100)으로 전환됨.
  현재 계정 보유 크레딧: **$120** (Free Tier $100 + 예산설정 활동 $20).
- 즉 "750시간/계정 합산" 제약(ADR 0001에서 2대 불가 사유)은 **이 계정엔 적용되지 않음**.
  크레딧 모델에서는 2번째 인스턴스도 그냥 크레딧을 더 소모할 뿐 약관 위반이 아님(동일 계정·동일 명의).

## 검토한 선택지

| 선택지 | 비용 | 비고 |
|---|---|---|
| NCP (ADR 0001) | 크레딧 10만원/3개월 후 과금 | 별도 가입·관리 필요 |
| **AWS 기존 계정에 EC2 2번째** | 크레딧에서 차감(≈월 $13~15/대) | **채택** — 계정·운영 일원화 |
| AWS RDS for MySQL | 추가 ≈월 $13 | **기각** — 사실상 인스턴스 한 대 값. 컨테이너로 대체 |

## 결정

1. dbvsdc 를 **사용자 AWS 계정의 신규 EC2 t3.micro** 한 대에 배포.
2. **이미지는 GitHub Actions(CI/CD)가 빌드 → GHCR 푸시**, EC2는 pull 후 실행만.
   - `ghcr.io/ppupy1209/dbvsdc-api`, `ghcr.io/ppupy1209/dbvsdc-web`
3. **RDS 미사용** — MySQL 을 같은 EC2의 **컨테이너**로 운영(추가 비용 0).
4. 1GB RAM 대비: **JVM 힙 상한(-Xmx384m) + 스왑 2GB**(별도 요금 없음).
5. 단계 운영: 지금은 `sample` 프로필(DB 불필요)로 api+web 만 → 실데이터 단계에서 MySQL 컨테이너 추가.

운영 실행: [docker-compose.prod.yml](../../docker-compose.prod.yml), 절차: [docs/DEPLOY.md](../DEPLOY.md).

## 결과 / 트레이드오프

- (+) 계정/운영 일원화(chipthrone과 같은 콘솔), 빌드는 CI가 처리해 서버 부하 없음.
- (+) RDS 미사용으로 월 ~$13 절감, 크레딧 수명 연장.
- (−) 영구 무료 아님 — 크레딧 소진(1대 ≈ 9개월, 2대 ≈ 4.5개월) 후 과금. 활동 보상으로 $200까지 연장 가능.
- (−) 1GB 단일 서버에 다중 컨테이너 → RAM 빡빡(힙 상한+스왑으로 완화). 한계 도달 시 RDS 분리 또는 인스턴스 상향.
- 비상시 이전 경로: 코드/이미지가 이식성 있어 NCP·Oracle Always Free 로 재배포 가능.

## 부수 결정

- 백엔드 빌드툴 **Maven → Gradle** 전환(chipthrone과 통일, wrapper 8.14.3). Docker 빌드로 검증 완료.
- 프론트 **Vite 전환 안 함** — SEO(검색 유입) 우위로 Next.js 유지. 추후 비용 최적화는 `output: export`(정적+nginx) 검토.
