# DATA-SOURCES — 데이터 소스

> ⚠️ "예상 수익률"을 주는 API는 없다. 받는 건 **과거 실적/기준 데이터**뿐. 미래 모드는 과거 최악 동일기간 경로를 재사용하는 보수적 스트레스 시나리오다.

## ★ 핵심 결정 (2026-06-24): 지수 연간 수익률을 큐레이션해 보유 — API 키 불필요

백테스트 핵심 데이터는 **각 지수(S&P500·나스닥·다우·코스피·코스닥)의 연간 등락률**이다.
- ETF 시세 API로 실시간 수집하는 방식은 **폐기**. 이유: ETF는 상장 역사가 짧고(2020 전후), 환율·API 키·일배치 운영 부담이 큼.
- 대신 **지수 연간 총수익률 근사치(배당 재투자 포함·현지통화·비용 차감 전)** 를 정적으로 보유. 역사가 길고(1995~), 키·배치 불필요. 매년 1행만 추가.
- 보유 위치: 프론트 `web/src/lib/indexData.ts` (단일 진실원), 백엔드는 동일 데이터를 MySQL에 시드해 `live`로 서빙.
- 출처: S&P500 = 연간 총수익률, 나스닥/다우/KOSPI/KOSDAQ = 가격수익률 + 평균 배당수익률 근사. KRX TR/실제 ETF 데이터로 정밀화 예정.
- ⚠️ 해외 지수는 USD 기준 → 원화 환산 시 환율 영향. 현재 미반영(교육용). 향후 원화 환산 옵션 검토 가능.

## 지수/ETF 백테스트용 데이터 (30/70 시뮬레이터)

위험자산 70%에 담는 인기 지수와, DC가 실제로 담는 국내 상장 추종 ETF.

| 지수 | 대표 국내 추종 ETF(예) | 비고 |
|---|---|---|
| S&P 500 | TIGER 미국S&P500 등 | 환율 반영 필요 |
| 나스닥100 | KODEX 미국나스닥100 등 | 환율 반영 필요 |
| 다우존스 | TIGER 미국다우존스30 등 | 환율 반영 필요 |
| 코스피 | KODEX 200 등 | 원화 |
| 코스닥 | KODEX 코스닥150 등 | 원화 |

데이터 소스(시세 → 수익률 계산):
- [공공데이터포털 – 증권상품시세정보(ETF/ETN)](https://www.data.go.kr/data/15094806/openapi.do) — KRX 제공
- [KRX OPEN API](https://openapi.krx.co.kr/)
- [한국투자증권 KIS Developers](https://apiportal.koreainvestment.com/apiservice) — 무료 REST, 단 계좌+앱키 필요
- [공공데이터포털 – 주식시세정보](https://www.data.go.kr/data/15094808/openapi.do) (보조)

⚠️ **주의**
- "예상 수익률" API는 없음 → 과거 시세로 트레일링 수익률 계산, 미래는 사용자 가정 또는 과거 스트레스 경로로만 투영.
- 해외 지수는 **환율(원/달러)** 영향. 국내 추종 ETF를 쓰면 환율·보수가 자동 반영(권장). 현재 시뮬레이터는 원지수 현지통화 수익률에 보수적 비용 차감만 적용하며, 원화 환산은 별도 과제로 남김.

## 사용 전략

- **MVP 단계**: 실시간 연동 없이, 지수 과거 수익률·대표 상품을 정적 데이터로 시작(현재 프론트 시안 = 예시값).
- **2차**: 키 발급 쉬운 공공데이터포털(증권상품시세정보)부터 일배치 연동 → MySQL 캐싱. 통합연금포털/KIS는 승인·계좌 절차 후 추가.

## 키/시크릿 관리 규칙

- API 키는 **절대 git에 커밋하지 않는다.** 백엔드 환경변수(NCP)로 주입.
- 키 발급처·발급일·호출 제한(쿼터)을 이 문서 표에 기록해 둘 것.
