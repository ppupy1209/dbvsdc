## 수정 1 — 미래 시뮬레이션 30년 투영 (국내 지수)

문제: 기간 슬라이더 상한이 과거/미래 공통으로 지수별 과거 데이터 길이에 묶여 KOSPI200=16년·KOSDAQ150=10년까지만 미래 투영 가능 → 커리어 전체(30년) 시뮬레이션 불가.

- `Simulator.tsx`: `maxPeriod`를 모드별로 분리. 과거=`min(30, 가용연수)`, **미래=30 고정**.
- `calc.ts` `simulate` 미래 분기 분리:
  - **평균 시나리오**: CAGR 반복이라 과거 길이와 무관 → `n = yearsInput`(최대 30) 전체 투영.
  - **최악 시나리오**: 실제 과거 윈도우 필요 → `n = min(yearsInput, 가용연수)`, 반환된 윈도우 길이로 `n` 보정. UI는 기존 `(과거 최악 N년 경로: start~end)` 라벨로 단축 기간 표기.
- 검증: 미래+KOSDAQ150+30년 → 평균 차트 31점(~30년), 최악 차트 11점(~10년), NaN 없음. 과거 백테스트는 KOSDAQ 10·KOSPI 16 제한 유지.
- 결정 근거: 사용자 선택 "미래는 30년 허용(평균만)". 토글 전환 시 평균=30년, 최악=가용기간으로 horizon이 달라지는 것은 의도된 동작.

### 배당/TR 조사 — 결론: nq·dj 교체 불필요, ks·kq 자동수집 불가

- 각 지수 기준 점검: 진짜 실측 TR은 S&P500뿐. nq·dj는 가격수익률+고정배당 근사, ks=가격+1.8%, kq=KODEX ETF 조정종가 프록시(기준 혼입).
- Codex 위임으로 KRX TR 자동 수집 시도 → **pykrx 환경 문제 + KRX CSV 호출 타임아웃 없이 hang**(~28분). 종료·롤백. (이때 Codex가 nq/dj를 깨진 값으로 7개 파일에 써둔 것을 발견 → 전부 HEAD로 복원 후 비용 작업만 재적용.)

- 추종 ETF(QQQ·DIA) 배당재투자 수익률과 대조: nq·dj 연도별 차이 **대부분 ±0.5%p**(ETF는 보수 net이라 gross 지수면 더 가까움) → 실측 TR 교체는 정확도 개선 미미 + ETF프록시 기준 혼입 + 이력 단축. **현 근사 유지** 결정.
