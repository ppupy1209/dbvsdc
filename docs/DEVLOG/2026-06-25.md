# DEVLOG 2026-06-25 — Docker/CI·CD 정비, 인프라 AWS 전환

## 한 일

chipthrone 저장소의 Docker/CI·CD 세팅을 참고해 dbvsdc에 맞게 이식. 동시에 배포 인프라를 NCP → AWS로 변경.

### Docker (로컬/운영 분리)
- **로컬**: [docker-compose.yml](../../docker-compose.yml) — `docker compose up --build` 하나로 mysql + api + web 전체 기동. 실제 기동·응답 검증 완료(api 8090 JSON 200, web 3000 HTTP 200).
- **운영**: [docker-compose.prod.yml](../../docker-compose.prod.yml) (신규) — 로컬 빌드 대신 **GHCR 이미지 pull** 실행. restart 정책, `JAVA_OPTS=-Xmx384m`, MySQL 컨테이너(RDS 미사용).
- [web/Dockerfile](../../web/Dockerfile) — Next.js standalone 프로덕션 멀티스테이지(+ `next.config.ts`에 `output:"standalone"`).
- 개발 전용(web-dev/Dockerfile.dev/compose 프로필)은 한 번 넣었다가 **복잡도 때문에 제거** — 상태를 "로컬 1개 + 운영 1개"로 단순화.

### 백엔드 빌드툴 Maven → Gradle
- `api/pom.xml` 삭제 → `build.gradle`/`settings.gradle`/gradle wrapper(8.14.3, chipthrone 것 재사용).
- [api/Dockerfile](../../api/Dockerfile): Gradle bootJar 멀티스테이지 + `JAVA_OPTS` 주입형 실행. plain jar 비활성화.
- `.dockerignore`/`.gitignore` target/ → build/·.gradle/ 로 갱신.
- **Docker 빌드로 컴파일까지 검증(BUILD SUCCESSFUL)**. ⚠️ 단, test는 `-x test`로 스킵 — `./gradlew build`의 테스트가 MySQL 없이 통과하는지는 **Codex 검증 필요**.

### CI / CD ([.github/workflows](../../.github/workflows))
- `ci.yml` — 푸시·PR 마다: api `./gradlew build` + web `npm run build`.
- `deploy.yml` — main 푸시 시 GHCR 푸시: `dbvsdc-api`, `dbvsdc-web` (`:latest` + `:<sha>`).

### 인프라 결정 변경 → [ADR 0002](../DECISIONS/0002-infra-aws-docker.md)
- NCP → **AWS 기존 계정(chipthrone)에 EC2 t3.micro 2번째**. 프리티어가 크레딧 모델($120)이라 2대도 약관 위반 아님.
- **RDS 미사용** — MySQL 컨테이너로 운영(월 ~$13 절감).
- 1GB 대비: JVM 힙 상한 + 스왑 2GB(무료). 절차는 [docs/DEPLOY.md](../DEPLOY.md)(신규).

## 결정 메모

- 프론트 **Vite 전환 안 함** — SEO 우위로 Next.js 유지. 비용 최적화는 추후 `output:export`(정적+nginx).
- 두 번째 EC2 비용 ≈ 월 $13~15(컴퓨팅+공인 IPv4). 크레딧 1대 ≈ 9개월 / 2대 ≈ 4.5개월.

## 남은 것 / 주의

- **Codex**: `./gradlew build` 테스트가 DB 없이 통과하는지 확인(CI 빨개질 수 있음).
- 미커밋 변경 2건(`web/src/components/Simulator.tsx`, `web/src/lib/indexData.ts`)은 이번 인프라 작업과 무관 → 별도로 둠. 특히 `indexData.ts` 주석 `출처:` → `출처`로 깨진 부분은 **실수로 보임, 확인 필요**.
- 실제 EC2 프로비저닝/도메인/HTTPS(리버스 프록시)는 미진행.
