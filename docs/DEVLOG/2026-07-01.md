# DEVLOG 2026-07-01 — 미래 Monte Carlo 확률 밴드 추가 → 사용자 판단으로 제거

## 요약

미래 시뮬레이션에 **Monte Carlo 확률 밴드**를 구현·검증했으나, **사용자 판단("확률 밴드 개념이
일반 사용자에게 어렵다")에 따라 같은 세션에서 제거**함. 코드는 원상복구(revert), 결정만 기록.

## 무엇을 만들었었나 (참고용 — 되돌림)

- `calc.ts monteCarloBands()`: 지수 실제 연도별 수익률을 IID 부트스트랩(1,500경로, mulberry32 고정시드=결정적),
  안전30/위험70+실부담비용 블렌딩 → p10/25/50/75/90 잔액 경로 + `probDcWins` + 종말 10~90% 범위 + 중앙값 CAGR.
- `simulate()` `"band"` 분기(중앙값 경로가 기존 verdict/세금 UI 구동, 이력<5년 평균 폴백).
- UI: 미래 3버튼(평균/확률 밴드/최악, 밴드 기본), 불확실성 콘(10~90%+25~75% 띠)+중앙값선,
  "N% 확률로 DC 우위 · 10~90% A~B", 방법 캡션, 호버 3줄, `scn=band` 공유.
- 검증: tsc 클린 · vitest 42/42 · build · 프리뷰 E2E(밴드 렌더·86% 확률·무회귀·모바일·다크) 통과했었음.

## 후속 — 미래 면책 문장 + 한글 줄바꿈 다듬기

밴드 제거로 재오픈된 #4를 **통계 개념 없이 쉬운 한 문장**으로 보완 + 전역 타이포 개선.

- **미래 면책 문장**: 미래 모드(평균/최악 **둘 다**) 상단 노트를 `"미래 수익은 해마다 크게 오르내려 확정이 아닙니다."`(굵게)로 시작하도록 교체. 뒤에 시나리오별 한 줄(평균=매년 같은 수익률 가정·손실구간 있음, 최악=과거 최악 경로·더 나쁠 수도)을 붙임. (기존 avgWarn은 평균에만 떴는데 이제 최악에도 노출)
- **한글 줄바꿈**: 본문 `text-wrap: pretty`(마지막 줄 고아 단어 방지) + 제목 `h1~h4 text-wrap: balance`(줄 길이 균형) + `.verdictText` balance. `word-break: keep-all`은 이미 있었음(어절 중간 안 끊김).
  - ⚠️ **함정**: globals.css에 넣으면 **Tailwind v4 Lightning CSS가 컴파일 때 text-wrap을 제거**함(모듈 CSS는 통과). → `layout.tsx`의 hoisted `<style precedence>`로 주입해 우회. globals.css엔 이유 주석만.
- 검증: tsc 클린 · vitest 35/35 · build 통과. 프리뷰: 면책 문장 평균/최악 양쪽 노출 확인, 계산된 `text-wrap-style`이 body=pretty·h1/h2=balance·footer=pretty로 적용, 모바일 375px 오버플로 0.

## 후속 2 — 신뢰성 UX 5종 (공유/PDF 제거·계산근거 모달·최악>평균 설명·환율 설명·유의사항 정리)

사용자 요청 5건 처리:
1. **공유/PDF 제거**: ‘결과 링크 복사’·‘PDF로 저장·인쇄’ 버튼 + `onShare`/`copied` 삭제. URL 하이드레이트/동기화(`shareState`)도 제거(공유 없이는 무의미 + 하이드레이트가 실제로 반쯤 깨져 있었음). `shareState.ts`·`shareState.test.ts` 삭제(테스트 35→30).
2. **‘계산 근거’ 모달**(신규 `MethodModal.tsx`+css): 결과 하단의 절제된 버튼("이 결과는 어떻게 계산되나요?")으로 열림. ① 데이터 출처 ② 계산 방식(+‘최악>평균’ Q&A) ③ 세금 계산(퇴직소득세 산식·IRP 30/40% 감면·예: 1억/10년≈426만) ④ 면책. 배경/ESC/✕ 닫기, 배경 스크롤 잠금, 모바일 바텀시트. **메인 화면은 버튼 한 줄만 차지.**
3. **‘최악’이 ‘평균’보다 높게 보이는 경우 설명**: 실측 재현 — KOSDAQ150 15년 = 최악 연평균 3.9% > 평균 3.3%(DC 8,853만 > 7,736만). 원인: ‘최악’은 *수익률 최저*가 아니라 *최종 적립금 최저* 구간이고, 실제 등락을 **안전자산 30%가 완충**해 ‘매년 같은 수익률’ 평균선보다 연평균이 높게 나올 수 있음(데이터 짧은 지수는 최악 구간=전체 기간이라 특히). 미래 노트 문구 재구성 + 모달 Q&A로 설명.
4. **원화 환산 설명 개선**: fxHint를 "환헤지 안 한 ETF처럼 지수 수익률+그해 원/달러 변동을 더해 원화 체감 수익으로 바꿈"으로, 추정 이유(World Bank 연평균이라 연말 급등락 일부만 반영)까지 명시.
5. **유의사항 중복 제거**: Footer 10항목 → 3항목(과거≠미래·추정 고지·자세한 건 ‘계산 근거’ 참조). 상세 방법론은 모달로 이관.

- 검증: tsc 클린 · vitest **30/30** · build 통과. 프리뷰(재시작한 새 서버 = 콘솔 오류 0): 버튼 제거·모달 열기/닫기(ESC·배경)·스크롤잠금·최악노트·fxHint·footer 3항목·모바일 바텀시트·오버플로 0.
- 주의: 편집 중 import를 파일 생성보다 먼저 저장해 dev 서버에 module-not-found 에러가 **누적**됐으나, build 통과 + 새 서버 콘솔 0으로 stale임을 확인.
