# ARCHITECTURE — dbvsdc

## 전체 구조

```
[사용자 브라우저]
      │
      ▼
  Vercel ──────────────  Next.js 프론트엔드 (Claude 담당)
  (무료)                 - 시뮬레이터 UI / 계산 로직(클라이언트)
      │                  - 정적 콘텐츠 / 개발기 블로그
      │ HTTPS REST (필요한 데이터만)
      ▼
  NCP 서버 ────────────  Spring Boot REST API (Codex 담당)
  (Java/Spring)          - 외부 공공데이터 API 프록시 + 캐싱
      │                  - 개발기/통계/공유결과 저장 (선택)
      ▼
  MySQL ───────────────  캐싱 스냅샷 / 콘텐츠 / 통계
```

## 설계 원칙

1. **계산은 가능한 한 클라이언트에서.** DB/DC 시뮬레이션은 순수 수식이라 브라우저에서 즉시 계산 → 서버 부하·비용 0, 응답 즉각적.
2. **백엔드는 "데이터·프록시·저장"만.** 외부 API 키 숨김, 호출량 절약(캐싱), 영속 데이터.
3. **외부 API는 실시간 호출 금지, 배치 캐싱.** 공공 API는 느리고 호출 제한·스키마 변동이 있음 → 일배치로 MySQL에 스냅샷 저장 후 서빙.
4. **DB는 최소 사용.** 정말 필요한 것만(개발기는 git 마크다운으로 대체 가능).

## 시뮬레이션 계산 모델 (초안 — 확정 필요)

> ⚠️ 아직 수식 미확정. 아래는 방향성. 확정 시 이 섹션 갱신 + ADR.

- **DB형 예상 수령액** ≈ (퇴직 직전 평균임금) × (근속연수)
  - 입력: 현재 연봉, 연봉상승률, 근속연수, (임금피크 여부)
- **DC형 예상 수령액** ≈ Σ(매년 적립금 × (1+운용수익률)^남은기간)
  - 입력: 연봉, 연봉상승률, 근속연수, 연 운용수익률(슬라이더)
- **손익분기 수익률**: DB 수령액 = DC 수령액이 되는 연 수익률 r* 를 역산
- 반영 요소(2차): 퇴직소득세, 운용보수, IRP 이전
