# ARCHITECTURE — dbvsdc

## 전체 구조

```
[사용자 브라우저]
      │
      ▼
  Vercel ──────────────  Next.js 프론트엔드 (Claude 담당)
  (무료)                 - 시뮬레이터 UI / 계산 로직(클라이언트)
      │                  - 정적 콘텐츠 / 개발기 블로그
      │ HTTPS REST (필요한 데이터만)
      ▼
  NCP 서버 ────────────  Spring Boot REST API (Codex 담당)
  (Java/Spring)          - 외부 공공데이터 API 프록시 + 캐싱
      │                  - 개발기/통계/공유결과 저장 (선택)
      ▼
  MySQL ───────────────  캐싱 스냅샷 / 콘텐츠 / 통계
```

## 설계 원칙

1. **계산은 가능한 한 클라이언트에서.** DB/DC 시뮬레이션은 순수 수식이라 브라우저에서 즉시 계산 → 서버 부하·비용 0, 응답 즉각적.
2. **백엔드는 "데이터·프록시·저장"만.** 외부 API 키 숨김, 호출량 절약(캐싱), 영속 데이터.
3. **외부 API는 실시간 호출 금지, 배치 캐싱.** 공공 API는 느리고 호출 제한·스키마 변동이 있음 → 일배치로 MySQL에 스냅샷 저장 후 서빙.
4. **DB는 최소 사용.** 정말 필요한 것만(개발기는 git 마크다운으로 대체 가능).

## 시뮬레이션 계산 모델

> ⚠️ 프론트 시안에 구현된 임시 모델. 법정 산식 근거 확인 후 정밀화 필요(아래 "확정 필요" 참조).

### 공통 입력
- 현재 연봉, 연봉상승률, 기간(1~30년), (임금피크 여부 — 2차)
- 안전자산 금리: **2026년 기준 연 3.0% 고정**(원리금보장형 가정). 매년 갱신 포인트.

### DB형 예상 수령액
- 법정 산식: **30일분 평균임금 × 근속연수** (근로자퇴직급여보장법)
- 평균임금 = 직전 3개월 임금총액/총일수 (상여·수당 포함 → 통상 월급보다 큼)
- 모델 근사: 월 평균임금 ≈ 현재연봉 × (1+상승률)^(연차-1) / 12 (연봉이 상여 포함이라고 가정)
- 수익률과 무관. 연봉상승률이 높을수록 유리.
- **임금피크제 옵션**: DB는 *퇴직 직전* 평균임금 기준이라, 임금피크·강등 시 최종임금이 깎이면 DB 수령액이 직접 감소(DC는 이미 적립돼 영향 없음). 시안은 최종 평균임금 30% 감액으로 가정(`PEAK_CUT`).

### DC형 예상 수령액 — 30/70 구성
- 안전자산 30%(예금 3.0%) + 위험자산 70%(지수 바스켓). **70%는 DC 법정 위험자산 한도**.
- 매년 부담금 = 월 임금(연 임금총액의 1/12). 매년 30/70 리밸런싱 가정.
- 잔액 점화식(보수적 연말 납입): `bal = bal × (1 + 혼합수익률) + 부담금`
- Blended return = `0.3 * depositRate + 0.7 * selectedIndexReturn`. The old global cost haircuts were removed.

### 두 가지 모드
1. **과거 백테스트**: 최근 N년의 *실제 연도별* 지수 수익률을 그대로 적용 → 하락장(2008·2022) 변동까지 표시. 연평균 수익률(CAGR)·최악의 해 함께 제공.
2. **미래 예측**: 두 시나리오를 제공. `평균`은 선택 지수의 과거 CAGR을 매년 적용하고, `최악`은 동일 기간 과거 구간 중 DC 최종 잔액이 가장 낮은 경로를 사용해 하락장·횡보장 순서위험을 보여줌.

### 세금 모델 (퇴직소득세)
현행 퇴직소득세 산식 기반으로 DB/DC 수령액에 적용:
1. 근속연수공제 → 환산급여 = (퇴직급여 − 공제)/근속연수 × 12
2. 환산급여공제 → 과세표준
3. 산출세액 = 누진세율 적용 ÷ 12 × 근속연수
- **IRP 연금수령 혜택**: 15년 균등수령 가정. 10년차까지 이연퇴직소득세의 70%, 11년차부터 60%를 적용.
- 과세 범위(확인됨): DC의 **회사부담금 + 그 운용수익 전액이 퇴직소득세 대상**(일시금) → 시안 모델이 현행과 일치. 연금수령 시 퇴직소득세의 70%/60%.
- 미반영(범위 밖): **개인 추가납입분**(IRP 세액공제 받은 원금·운용수익)은 일시금 시 기타소득세 16.5%, 연금 시 연금소득세 3.3~5.5%로 별도 과세. DB vs DC(회사 제공 퇴직급여) 비교에는 불필요하나, 향후 IRP 통합 계산 시 분리 필요.

### 정확도 한계 (사용자에게 고지 — UI 반영 완료)
- 지수 수익률은 예시값 → 실데이터는 추종 ETF로 대체
- 해외 지수 **환율(원/달러)** 미반영
- FX, management fees, trading costs, tracking difference, and hedge costs are not currently modeled; add them later as explicit product/FX variables.
- 세금은 추정치, 개인별 조건 따라 상이
- 미래 예측은 평균(CAGR) 시나리오와 과거 최악 동일기간 시나리오를 제공하지만, 둘 다 미래 보장은 아님
