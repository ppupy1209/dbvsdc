# PROJECT — dbvsdc

## 한 줄 정의

퇴직연금 **DB형(확정급여) vs DC형(확정기여)** 을 사용자 조건으로 비교·시뮬레이션하는 웹 서비스.

## 왜 만드나 (문제)

- 직장인 대부분이 자기 퇴직연금이 DB인지 DC인지, 어느 쪽이 유리한지 모른다.
- 특히 "DC로 바꾸면 이득인가?"는 **운용 수익률 가정**에 따라 답이 갈리는데, 이걸 직관적으로 보여주는 도구가 없다.
- 핵심 질문: **"내 조건에서 수익률 몇 %부터 DC가 DB를 역전하는가?"** (손익분기 수익률)

## 타깃 사용자

- 한국 직장인 (DB/DC 선택권이 있거나, 전환을 고민 중인 사람)
- 금융 지식이 깊지 않아도 슬라이더 몇 번으로 결과를 보게 한다.

## 핵심 가치 / 차별점

1. **손익분기 수익률** 표시 — "몇 %부터 DC가 유리한가" (사람들이 진짜 궁금한 지점)
2. **시나리오 비교** — 낙관/중립/비관 동시 표시
3. **클라이언트 계산 중심** — 빠르고, 서버 비용 거의 0
4. **결과 공유** — URL 파라미터 인코딩으로 서버 부담 없이 공유
